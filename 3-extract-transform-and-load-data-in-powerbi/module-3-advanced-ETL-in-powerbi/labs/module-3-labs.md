### ðŸ§ª Lab 2: Import a Dataset with Best Practices
* **Objective:** Import a large sales dataset while adhering to performance best practices. The task involved filtering data *during* the import process to minimize memory footprint.
* **Files:**
    * [ðŸ“‚ View Lab Files](./lab-2-import-a-dataset-best-practices/)
    * [ðŸ“Š View Optimized Model](./lab-2-import-a-dataset-best-practices/Optimized_Sales_Import.pbix)

#### Key Actions Performed:
* **Delimited Text Import:** Configured the CSV import settings to use **Tab** delimiters, ensuring the raw text parsed correctly into columns.
* **Query Folding (Filtering at Source):** Applied a **Date Filter** ("In the Previous 5 Years") immediately upon import. This reduces the volume of data loaded into the model, significantly improving refresh performance.
* **Proactive Cleaning:** Removed rows with **Errors** in the `ProductKey` column and filtered out **Null** values in the `OrderDate` column before any transformation logic was applied.

### ðŸ§ª Lab 4: Disabling Helper Queries
* **Objective:** Optimize the data model by preventing intermediate "Helper Queries" from loading into memory. This ensures only the final, clean data is accessible to report users.
* **Files:**
    * [ðŸ“‚ View Lab Files](./lab-4-disabling-helper-queries/)
    * [ðŸ“Š View Tuned Model](./lab-4-disabling-helper-queries/Disabling%20Helper%20Queries.pbix)

#### Key Actions Performed:
* **Identification:** Identified intermediate queries (e.g., "Transform Sample File") that were automatically generated by Power BI during a folder import.
* **Load Management:** Right-clicked specific queries and unchecked **"Enable Load"**. This keeps the query active for transformation logic but stops it from consuming RAM or appearing in the final report view.
* **Impact:** Reduced the visual clutter in the "Fields" pane and improved report refresh speed.
